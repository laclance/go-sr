# Security Policy

## Supported Versions

Only the current `main` branch and latest GitHub release are supported for security fixes unless this repo documents a broader support window later.

## Reporting a Vulnerability

Please do not disclose vulnerabilities publicly before a fix is available.

Use GitHub private vulnerability reporting if it is enabled for this repository. If it is not enabled, open a minimal security advisory request in GitHub without exploit details, and the maintainer will coordinate next steps.
